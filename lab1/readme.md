# Lab1 — ระบบธนาคารขยะ (Trash Bank Web)

คำอธิบาย
ระบบเว็บสำหรับจัดการ "ธนาคารขยะ" เพื่อบันทึกและติดตามการรับ-ส่งของเสีย โดยมีการแจ้งเตือนและระบบลงชื่อเข้าใช้ 2 ระดับ (ผู้จัดการ และนักปฏิบัติการ)

คุณสมบัติหลัก
- ลงทะเบียนรายการขยะและปริมาณ
- ระบบแจ้งเตือนและสถานะการดำเนินงาน
- สิทธิ์การใช้งาน 2 ระดับ: ผู้จัดการ / นักปฏิบัติการ
- รองรับการใช้งานบนมือถือและเดสก์ท็อป

โครงสร้างไฟล์ (File Structure)

โฟลเดอร์หลักและไฟล์เอกสารภายในโปรเจคตอนนี้:

```
lab1/
├─ ARCHITECTURE.md    — สถาปัตยกรรมระบบ และแผนการ deploy
├─ CLAUDe.md          — เอกสารเสริม (CLAUDe)
├─ readme.md          — ไฟล์นี้: คำอธิบาย และโครงสร้างโปรเจค
└─ SRS.md             — ข้อกำหนดซอฟต์แวร์ (Software Requirements Specification)
```

คำอธิบายไฟล์ย่อ ๆ
- `ARCHITECTURE.md`: อธิบายสถาปัตยกรรมระบบ, ส่วนประกอบหลัก, และการตั้งค่า environment
- `CLAUDe.md`: เอกสารที่เกี่ยวข้องหรืออ้างอิงเฉพาะโครงการ
- `readme.md`: คู่มือนี้ — รวมโครงสร้างโปรเจคและคำแนะนำการใช้งานเบื้องต้น
- `SRS.md`: ระบุความต้องการของระบบและข้อกำหนดเชิงฟังก์ชัน/ไม่เชิงฟังก์ชัน

โครงสร้างที่แนะนำ (หากเพิ่มซอร์สโค้ดในอนาคต)

```
lab1/
├─ src/               — โค้ดต้นทาง (backend, frontend)
│  ├─ backend/
│  └─ frontend/
├─ tests/             — ชุดทดสอบ
├─ docs/              — เอกสารเพิ่มเติม
├─ scripts/           — สคริปต์ช่วยติดตั้ง/รัน
├─ .gitignore
└─ LICENSE
```

การติดตั้ง (ตัวอย่างทั่วไป)
- ติดตั้ง dependencies ที่จำเป็น (ขึ้นกับเทคโนโลยีที่ใช้)
- กำหนดค่าแวดล้อมตามเอกสารใน ARCHITECTURE.md และ SRS.md
- ย้ายหรือสร้างฐานข้อมูล (ตัวอย่าง: SQLite) ตามที่กำหนด

การใช้งาน
- อ่านรายละเอียดและวิธีใช้งานเพิ่มเติมใน SRS.md และ ARCHITECTURE.md

การร่วมพัฒนา
- Fork หรือคัดลอก repo นี้ แล้วแก้ไขเอกสารหรือโค้ด
- ส่ง PR พร้อมคำอธิบายการเปลี่ยนแปลง

ผู้ติดต่อ
- ดูข้อมูลผู้พัฒนา/ผู้รับผิดชอบในเอกสารโครงการหรือไฟล์ติดต่อที่แนบมา

ลิขสิทธิ์ / ใบอนุญาต
- ระบุประเภทใบอนุญาตตามต้องการ (เช่น MIT, Apache-2.0) ในไฟล์ LICENSE

หมายเหตุ
- เอกสารอื่น ๆ ภายในโฟลเดอร์ (ARCHITECTURE.md, SRS.md) ควรอัปเดตให้สอดคล้องกับ README นี้
